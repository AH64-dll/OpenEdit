# v1.2 Follow-up: CI infrastructure for sandbox tests — RESOLVED

**Status:** RESOLVED in v1.2.0 (release tag `v1.2.0`).

## What was done

- Added `.github/workflows/ci.yml` with 3 jobs:
  - `python-unit` — 308 Python unit tests on Python 3.11/3.12/3.14
  - `rust-build` — `cargo build` + `cargo test` on the Rust sandbox
  - `bwrap-tests` — un-ignores the 5 Rust integration tests and 5 Python e2e tests written in v1.1 T3; uses `bin/setup_render_cgroup.sh` with sudo to set up cgroup v2
- The 5 Rust tests + 5 Python e2e tests now run in CI on every push to main, every PR, and every `v*` tag.
- The tests stay ignored/skipped locally (where bwrap + cgroup v2 may not be available) but run automatically in CI.

## Acceptance

- ✅ All 5 `#[ignore]`-d Rust tests are un-ignored in CI (via `cargo test --features integration -- --include-ignored`)
- ✅ All 5 skipped Python e2e tests run in CI (the skip condition checks for bwrap, which is present in the bwrap-tests job)
- ✅ CI pipeline runs them on every PR

## Trade-off accepted

- The bwrap-tests job requires passwordless sudo (to set up the cgroup v2 directory). GitHub-hosted `ubuntu-latest` runners have this. Self-hosted runners would need to grant it explicitly.
- ~5-10 min per bwrap-tests run. Canceled automatically on PR pushes (concurrency: cancel-in-progress on PRs).
