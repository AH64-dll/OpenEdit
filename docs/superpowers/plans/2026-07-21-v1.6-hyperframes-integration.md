# v1.6 — HyperFrames HTML Overlay Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend `trigger_render` with a `mode: "overlay"` path that composites an HTML overlay layer (rendered by `@hyperframes/engine`) over the existing MLT background video via ffmpeg, producing a single MP4 the v1.5 visual verification loop samples and judges. The composited path is **additive** — existing renders (no overlays, or `mode != "overlay"`) are byte-identical to v1.5 behavior.

**Architecture:** A new `open_edit.serve.html_overlay` module (~250 lines) with 4 public functions and 1 exception class:
- `generate_composition_html(timeline, project_workdir, render_spec) -> str` — pure HTML generator (escapes `{{key}}` substitution, applies track assignment, bakes values into the HTML).
- `render_overlay_layer(comp_html_path, output_path, render_spec, should_cancel=None) -> Path` — subprocess wrapper for `hyperframes render` with verified CLI flags; raises `OverlayRenderError` on failure.
- `composite_with_background(bg_path, overlay_path, output_path, render_spec, should_cancel=None) -> Path` — subprocess wrapper for ffmpeg's `overlay` filter with explicit audio mapping.
- `render_composited(timeline, project_workdir, render_spec, bg_renderer, should_cancel=None) -> Path` — async orchestrator running bg render + composition HTML gen + overlay render concurrently; raises `OverlayRenderError(bg_path=...)` on failure.

A new `package.json` at the repo root pins `hyperframes@0.7.65` exactly. `pi_bridge._run_trigger_render` gains a 5-line dispatch to `html_overlay.render_composited` when `mode == "overlay"` AND the project has overlays. Everything else is byte-identical to v1.5.

**Tech Stack:** Python 3.11+, stdlib only for `html_overlay` (no new deps), `ffmpeg` + `ffprobe` shell-outs (already used by v1.5), `node` 18+ + `hyperframes@0.7.65` (pinned via `package.json`).

**Spec:** `docs/superpowers/specs/2026-07-21-v1.6-hyperframes-integration-design.md`

## Global Constraints

- Python 3.11+; matches the rest of `open_edit`.
- All `subprocess` calls (hyperframes CLI, ffmpeg, ffprobe) use argv-list, **never** `shell=True`. Security-critical; the LLM can supply values that end up in the argv.
- `RenderSpec` (TypedDict) is the single source of truth for `{width, height, fps, duration_sec, mode, hyperframes_bin, hyperframes_timeout_s, tmpdir}`. Built once in `pi_bridge` from the MLT project profile, passed to all 4 stages of the pipeline.
- All disk paths absolute; project-relative paths forbidden in subprocess args.
- `OverlayRenderError(message, bg_path=None)` is the only exception the orchestrator raises. `bg_path` is set when the bg render succeeded but a downstream step failed — `pi_bridge` returns `e.bg_path` instead of re-rendering the bg.
- Test convention: pytest, fixtures from `tests/conftest.py`, isolated `tmp_path` per test. No real subprocesses in unit tests — all subprocess calls are mocked.
- Commit style: `[open_edit] v1.6 t<N>: <message>` (matches the 12 v1.5 commits and 11 v1.4 commits on main).
- No new Python runtime dependencies — stdlib only for `html_overlay` (only new dep is `hyperframes@0.7.65` for the render side, pinned in `package.json`).
- v1.6 new env vars (defaults in `serve_env.py`): `OPEN_EDIT_HYPERFRAMES_BIN=node_modules/.bin/hyperframes`, `OPEN_EDIT_HYPERFRAMES_TIMEOUT_SECONDS=120`, `OPEN_EDIT_OVERLAY_TMPDIR=` (empty = project-scoped `<project>/.open_edit/tmp/overlay/`).
- v1.6 CLI flags (verbatim from spec §16, verified against `hyperframes@0.7.65`): positional `[DIR]` at the end, `-c compositions/overlay.html` (not `--input`), `-f $FPS` (not `--fps`), `-o $out`, `-q standard`, `--strict`. No `--width`/`--height`/`--transparent` (they don't exist).
- v1.6 composition HTML structure (verbatim from spec §4, verified against the engine's canonical example): root `<div id="root" data-composition-id="..." data-start="0" data-duration="<total>" data-width="..." data-height="..." data-fps="..." data-no-timeline>`. Each clip `<div id="overlay_<hash>" class="clip" data-start="..." data-duration="..." data-track-index="...">`. No `<script>` tag, no `window.__timelines` registration, no `window.__open_edit_vars` blob.
- v1.6 ffmpeg composite: `ffmpeg -y -i bg.mp4 -i overlay.mov -filter_complex "[0:v][1:v]overlay=eof_action=pass" -map 0:a -map [outv] -c:a copy -c:v libx264 final.mp4`. `-map 0:a -map [outv] -c:a copy` is explicit (don't let ffmpeg auto-select). `eof_action=pass` means: when the overlay (secondary `[1:v]`) reaches EOF first, the bg (primary `[0:v]`) continues unmodified.
- v1.6 disk preflight: 500 MB soft warn (logged, render proceeds), 2 GB hard cap (raises `OverlayRenderError`, `pi_bridge` falls back to bg).
- v1.6 cancellation: `should_cancel: Callable[[], bool]` plumbed from v1.5's agent loop WS-disconnect handler to all 3 subprocess stages (bg, hyperframes overlay, ffmpeg composite). Each subprocess polls `should_cancel` before and after the call; on True, kills the child process.
- Existing 566 tests must still pass after v1.6 (566 = v1.5 final + the 138 WIP tests, with the pre-existing `test_llm_stream_anthropic_surfaces_missing_api_key` deselected and 5 sandbox-skipped); new tests = 44; total ≈ 610.

```python
"""v1.6 HTML overlay compositing.

The composited render is a 4-stage pipeline that runs concurrently where
possible (spec §2):

    trigger_render({mode: "overlay"})
       │
       │  pi_bridge._run_trigger_render checks: overlays?  mode=="overlay"?
       │  → if both: composited path. Otherwise: existing MLT path (v1.5).
       │
       ├─ Stage 1: bg render              ─┐
       │  (existing _run_mlt_only_render)  │  concurrent
       │                                  │
       ├─ Stage 2: composition HTML gen   ─┤
       │  (generate_composition_html)     │
       │                                  │
       ├─ Stage 3: overlay render         ─┘
       │  (hyperframes render → overlay.mov, ProRes 4444)
       │
       └─ Stage 4: ffmpeg composite
            ffmpeg -i bg.mp4 -i overlay.mov
              -filter_complex "[0:v][1:v]overlay=eof_action=pass"
              -map 0:a -map [outv] -c:a copy
            → final.mp4

The v1.5 visual verification loop runs unchanged on `final.mp4`.

Public surface (all 4 functions + 1 exception):

  * :func:`generate_composition_html` — pure HTML generator
  * :func:`render_overlay_layer` — subprocess wrapper for hyperframes
  * :func:`composite_with_background` — subprocess wrapper for ffmpeg
  * :func:`render_composited` — async orchestrator
  * :exc:`OverlayRenderError` — raised on any failure (carries `bg_path`)

This module has zero new Python dependencies (stdlib only). The only
new binary dependency is `hyperframes@0.7.65` pinned in `package.json`.
"""
from __future__ import annotations

import logging
import os
from pathlib import Path

_LOG = logging.getLogger("open_edit.serve.html_overlay")


class OverlayRenderError(Exception):
    """Raised when the composited render fails.

    Carries ``bg_path`` when the bg render succeeded but a downstream step
    failed — lets ``pi_bridge``'s fallback reuse the completed bg without
    re-running the MLT encode (the bg encode is the slow part).
    """

    def __init__(self, message: str, bg_path: Path | None = None) -> None:
        super().__init__(message)
        self.bg_path = bg_path


def _resolve_hyperframes_bin() -> str:
    """Return the path to the hyperframes binary.

    Order of resolution (spec §5):
      1. ``OPEN_EDIT_HYPERFRAMES_BIN`` env var, if set — used verbatim.
      2. ``node_modules/.bin/hyperframes`` if it exists (set up by
         ``npm install`` at the repo root).
      3. Bare ``npx hyperframes`` (network resolution + version-drift risk;
         a WARNING is logged so the operator notices).

    Never raises.
    """
    env_bin = os.environ.get("OPEN_EDIT_HYPERFRAMES_BIN", "").strip()
    if env_bin:
        return env_bin
    pinned = Path("node_modules/.bin/hyperframes")
    if pinned.is_file():
        return str(pinned.resolve())
    _LOG.warning(
        "hyperframes pinned binary not found at %s; falling back to npx hyperframes "
        "(network resolution + version drift risk). Run npm install at repo root.",
        pinned,
    )
    return "npx hyperframes"


__all__ = [
    "OverlayRenderError",
    "_resolve_hyperframes_bin",
]
```

```python
# (existing imports + _LOG + OverlayRenderError + _resolve_hyperframes_bin stay)

import hashlib
import html
import re
from typing import Any, Iterable

from open_edit.ir.types import HtmlOverlay, Timeline  # noqa: F401  (re-exported)


# ---------------------------------------------------------------------------
# HTML composition generator (spec §4)
# ---------------------------------------------------------------------------

# Whitelist of variable types we inline at HTML-generation time. Non-primitive
# values raise OverlayRenderError in v1.6 (no JSON-blob support).
_PRIMITIVE_TYPES = (str, int, float, bool)


def _inline_variables(template_html: str, variables: dict[str, Any]) -> str:
    """Replace {{key}} placeholders in template_html with html.escape()'d values.

    Non-primitive values (dicts, lists, None) raise OverlayRenderError.
    Missing keys are left literal (logged at WARNING level).
    """
    def replace(match: re.Match) -> str:
        key = match.group(1).strip()
        if key not in variables:
            _LOG.warning("template references missing variable: %s", key)
            return match.group(0)
        value = variables[key]
        if not isinstance(value, _PRIMITIVE_TYPES):
            raise OverlayRenderError(
                f"non-primitive variable not supported in v1.6: key={key!r} type={type(value).__name__}"
            )
        return html.escape(str(value), quote=True)

    return re.sub(r"\{\{\s*([a-zA-Z_][\w\.]*)\s*\}\}", replace, template_html)


def _resolve_template_path(template_path: str, project_workdir: Path) -> Path:
    """Resolve a template path; reject absolute, parent-traversal, and symlink escapes.

    Lookup order: project_workdir/<template_path>, then built-in templates.
    Returns the resolved Path. Raises OverlayRenderError on rejection or not-found.
    """
    candidate_path = Path(template_path)
    if candidate_path.is_absolute():
        raise OverlayRenderError(f"absolute template_path not allowed: {template_path}")
    if any(part == ".." for part in candidate_path.parts):
        raise OverlayRenderError(f"template_path with '..' not allowed: {template_path}")

    project_candidate = (project_workdir / template_path).resolve()
    if project_candidate.is_file():
        return project_candidate

    builtin_candidate = (Path(__file__).parent / "templates" / "overlay" / template_path).resolve()
    if builtin_candidate.is_file():
        return builtin_candidate

    raise OverlayRenderError(f"template_not_found: {template_path}")


def _assign_tracks(overlays: list[HtmlOverlay]) -> list[tuple[HtmlOverlay, int]]:
    """Greedy non-overlapping track assignment.

    Sort by position_sec. For each overlay, find the lowest track index whose
    last overlay has already ended; if none exists, allocate a new track.
    """
    sorted_overlays = sorted(overlays, key=lambda o: (o.position_sec, o.id))
    track_end: list[float] = []  # track_end[i] = end time of the last overlay on track i
    result: list[tuple[HtmlOverlay, int]] = []
    for overlay in sorted_overlays:
        assigned = False
        for i, end in enumerate(track_end):
            if end <= overlay.position_sec:
                track_end[i] = overlay.position_sec + overlay.duration_sec
                result.append((overlay, i))
                assigned = True
                break
        if not assigned:
            track_end.append(overlay.position_sec + overlay.duration_sec)
            result.append((overlay, len(track_end) - 1))
    return result


def _clip_id(overlay: HtmlOverlay) -> str:
    """Deterministic, stable id for the clip <div>."""
    return f"overlay_{overlay.id}"


def generate_composition_html(
    timeline: Timeline,
    project_workdir: Path,
    render_spec: dict,
) -> str:
    """Generate the HyperFrames composition HTML for a Timeline's overlays.

    Pure function — no subprocess, no I/O. Returns a string ready to be
    written to <tmpdir>/compositions/overlay.html.

    Raises OverlayRenderError on:
      * template path validation failure (absolute, .., symlink escape, not found)
      * non-primitive variable in any overlay
    """
    overlays = list(timeline.overlays)
    if not overlays:
        # Defensive: callers should short-circuit before this (overlay mode requires
        # at least one overlay). Return an empty composition rather than crashing.
        overlays = []

    width = int(render_spec["width"])
    height = int(render_spec["height"])
    fps = int(render_spec["fps"])
    total_duration = float(render_spec["duration_sec"])

    track_assignment = _assign_tracks(overlays)

    clip_divs: list[str] = []
    for overlay, track_idx in track_assignment:
        template_path = _resolve_template_path(overlay.template_path, project_workdir)
        template_html = template_path.read_text(encoding="utf-8")
        inlined = _inline_variables(template_html, overlay.variables)
        clip_id = _clip_id(overlay)
        clip_divs.append(
            f'    <div id="{clip_id}" class="clip" '
            f'data-start="{overlay.position_sec}" '
            f'data-duration="{overlay.duration_sec}" '
            f'data-track-index="{track_idx}">\n'
            f'      {inlined.strip()}\n'
            f'    </div>'
        )

    clips_block = "\n".join(clip_divs) if clip_divs else ""
    return (
        f'<!doctype html>\n'
        f'<html lang="en">\n'
        f'<head>\n'
        f'  <meta charset="UTF-8">\n'
        f'  <meta name="viewport" content="width={width}, height={height}">\n'
        f'  <title>open_edit_overlay</title>\n'
        f'  <style>\n'
        f'    html, body, [data-composition-id] {{ margin: 0; padding: 0; background: transparent; }}\n'
        f'    html, body {{ width: {width}px; height: {height}px; overflow: hidden; }}\n'
        f'    .clip {{ position: absolute; inset: 0; }}\n'
        f'  </style>\n'
        f'</head>\n'
        f'<body>\n'
        f'  <div id="root"\n'
        f'       data-composition-id="open_edit_overlay"\n'
        f'       data-start="0"\n'
        f'       data-duration="{total_duration}"\n'
        f'       data-width="{width}"\n'
        f'       data-height="{height}"\n'
        f'       data-fps="{fps}"\n'
        f'       data-no-timeline>\n'
        f'{clips_block}\n'
        f'  </div>\n'
        f'</body>\n'
        f'</html>\n'
    )


__all__ = [
    "OverlayRenderError",
    "_resolve_hyperframes_bin",
    "generate_composition_html",
]
```

```python
# ---------------------------------------------------------------------------
# Subprocess wrappers (spec §5)
# ---------------------------------------------------------------------------

import subprocess  # for the subprocess.TimeoutExpired catch


def _estimate_overlay_size_mb(timeline: Timeline) -> int:
    """Estimate the overlay.mov size in MB.

    Rough heuristic per spec §5: 1 MB/s for static content (measured in
    spec §16 — 2.0s @ 1920x1080 of static content = 2.1 MB). For
    full-motion content this is a significant underestimate (~30 MB/s);
    the preflight is intentionally biased toward false-negatives on
    full-motion (we'd rather render and find out than false-positive
    block a 5-minute static title card).
    """
    total_seconds = sum(o.duration_sec for o in timeline.overlays)
    return int(round(total_seconds * 1.0))  # 1 MB/s


def _disk_footprint_check(estimated_mb: int, tmpdir: Path) -> None:
    """Log a WARNING if the estimate is large; raise OverlayRenderError if too large.

    Thresholds from spec §5: soft warn at 500 MB, hard cap at 2 GB.
    """
    if estimated_mb > 2048:
        raise OverlayRenderError(
            f"overlay_render_too_large: ~{estimated_mb} MB exceeds 2048 MB cap. "
            f"Shorten the overlay duration or render in segments."
        )
    if estimated_mb > 500:
        _LOG.warning(
            "overlay estimated ~%d MB; tmpdir=%s. This may take a while and use significant disk.",
            estimated_mb, tmpdir,
        )


def _kill_subprocess(proc: subprocess.Popen) -> None:
    """Best-effort subprocess kill; ignores errors (the proc may already be dead)."""
    try:
        proc.kill()
    except (ProcessLookupError, OSError):
        pass
    try:
        proc.wait(timeout=2)
    except subprocess.TimeoutExpired:
        pass


def render_overlay_layer(
    composition_html_path: Path,
    output_path: Path,
    render_spec: dict,
    should_cancel: "callable | None" = None,
) -> Path:
    """Run the hyperframes CLI to render `composition_html_path` to `output_path`.

    Returns the `output_path` on success. Raises `OverlayRenderError` on
    any failure (binary missing, timeout, non-zero exit). Polls
    `should_cancel` before and after the subprocess call; on True, kills
    the child process and raises OverlayRenderError (with no bg_path).
    """
    if should_cancel and should_cancel():
        raise OverlayRenderError("cancelled before overlay render")

    # Build the composition subdir inside the project tmpdir.
    tmp_project_dir = Path(render_spec["tmpdir"])
    comp_subdir = tmp_project_dir / "compositions"
    comp_subdir.mkdir(parents=True, exist_ok=True)
    # Mirror the composition HTML into the standard layout.
    target_html = comp_subdir / "overlay.html"
    target_html.write_text(composition_html_path.read_text(encoding="utf-8"), encoding="utf-8")

    # Verify CLI syntax (spec §16): positional [DIR], -c <html>, -f <fps>,
    # --format mov, -o <out>, -q standard, --strict.
    bin_argv = shlex.split(render_spec["hyperframes_bin"])
    cmd = bin_argv + [
        "render",
        "-c", "compositions/overlay.html",
        "-f", str(int(render_spec["fps"])),
        "--format", "mov",
        "-o", str(output_path),
        "-q", "standard",
        "--strict",
        str(tmp_project_dir.resolve()),
    ]

    _LOG.info("render_overlay_layer: %s", cmd)
    try:
        proc = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            timeout=int(render_spec["hyperframes_timeout_s"]),
            shell=False,
            check=False,
        )
    except FileNotFoundError as exc:
        raise OverlayRenderError(f"hyperframes binary not found: {exc}") from exc
    except subprocess.TimeoutExpired as exc:
        raise OverlayRenderError(
            f"hyperframes timed out after {exc.timeout}s"
        ) from exc

    if should_cancel and should_cancel():
        raise OverlayRenderError("cancelled after overlay render")

    if proc.returncode != 0:
        raise OverlayRenderError(
            f"hyperframes non-zero exit ({proc.returncode}): "
            f"stderr={proc.stderr.strip()[:500]}"
        )

    if not output_path.is_file() or output_path.stat().st_size == 0:
        raise OverlayRenderError(
            f"hyperframes produced no output: {output_path} (size={output_path.stat().st_size if output_path.exists() else 'missing'})"
        )

    return output_path


def composite_with_background(
    bg_path: Path,
    overlay_path: Path,
    output_path: Path,
    render_spec: dict,
    should_cancel: "callable | None" = None,
) -> Path:
    """Composite the overlay MOV over the bg MP4 via ffmpeg.

    Verifies the ffmpeg filter from spec §5: `[0:v][1:v]overlay=eof_action=pass`,
    explicit `-map 0:a -map [outv] -c:a copy` so bg's audio wins and isn't
    re-encoded. Returns the `output_path`. Raises OverlayRenderError on
    any failure (binary missing, timeout, non-zero exit).
    """
    if should_cancel and should_cancel():
        raise OverlayRenderError("cancelled before ffmpeg composite")

    cmd = [
        "ffmpeg", "-y",
        "-i", str(bg_path),
        "-i", str(overlay_path),
        "-filter_complex", "[0:v][1:v]overlay=eof_action=pass",
        "-map", "0:a",
        "-map", "[outv]",
        "-c:a", "copy",
        "-c:v", "libx264",
        str(output_path),
    ]

    _LOG.info("composite_with_background: %s", cmd)
    try:
        proc = subprocess.run(
            cmd,
            capture_output=True,
            text=True,
            timeout=int(render_spec["hyperframes_timeout_s"]),  # reuse timeout config
            shell=False,
            check=False,
        )
    except FileNotFoundError as exc:
        raise OverlayRenderError(f"ffmpeg binary not found: {exc}") from exc
    except subprocess.TimeoutExpired as exc:
        raise OverlayRenderError(
            f"ffmpeg composite timed out after {exc.timeout}s"
        ) from exc

    if should_cancel and should_cancel():
        raise OverlayRenderError("cancelled after ffmpeg composite")

    if proc.returncode != 0:
        raise OverlayRenderError(
            f"ffmpeg composite failed (exit {proc.returncode}): "
            f"stderr={proc.stderr.strip()[:500]}"
        )

    if not output_path.is_file() or output_path.stat().st_size == 0:
        raise OverlayRenderError(
            f"ffmpeg produced no output: {output_path}"
        )

    return output_path


__all__ = [
    "OverlayRenderError",
    "_resolve_hyperframes_bin",
    "generate_composition_html",
    "render_overlay_layer",
    "composite_with_background",
]
```

```python
# ---------------------------------------------------------------------------
# Async orchestrator (spec §6)
# ---------------------------------------------------------------------------

import asyncio
import shutil
import tempfile


async def render_composited(
    timeline: Timeline,
    project_workdir: Path,
    render_spec: dict,
    bg_renderer: "callable",
    should_cancel: "callable | None" = None,
) -> Path:
    """Concurrent composited-render pipeline.

    Runs the bg render, composition HTML gen, and overlay render concurrently
    where possible. The final ffmpeg composite waits for both the bg and
    overlay to complete.

    Returns the path to the final composited MP4.

    Raises:
        OverlayRenderError on any failure. The exception carries `bg_path`
        when the bg render succeeded but a downstream step failed — lets
        pi_bridge's fallback return the completed bg without re-encoding.
    """
    tmpdir = Path(render_spec["tmpdir"])
    overlay_cfg_tmpdir = render_spec.get("overlay_tmpdir")
    if overlay_cfg_tmpdir:
        tmpdir = Path(overlay_cfg_tmpdir)
    tmpdir.mkdir(parents=True, exist_ok=True)
    compositions_subdir = tmpdir / "compositions"
    compositions_subdir.mkdir(parents=True, exist_ok=True)
    comp_html_path = compositions_subdir / "overlay.html"

    bg_path_holder: dict[str, Path | None] = {"path": None}

    # Stage 1: bg render (in a thread, via bg_renderer).
    async def _bg():
        return await asyncio.to_thread(bg_renderer)

    # Stage 2: composition HTML generation.
    async def _gen_comp_html():
        return await asyncio.to_thread(generate_composition_html, timeline, project_workdir, render_spec)

    # Stage 3: overlay render (hyperframes subprocess).
    async def _overlay():
        return await asyncio.to_thread(
            render_overlay_layer, comp_html_path, tmpdir / "overlay.mov", render_spec, should_cancel,
        )

    # Stage 4: ffmpeg composite.
    async def _composite(bg_path: Path, overlay_path: Path):
        return await asyncio.to_thread(
            composite_with_background, bg_path, overlay_path, tmpdir / "final.mp4", render_spec, should_cancel,
        )

    try:
        if should_cancel and should_cancel():
            raise OverlayRenderError("cancelled before render", bg_path=None)

        # Pre-flight: disk footprint check (uses timeline, doesn't need a render).
        estimated_mb = _estimate_overlay_size_mb(timeline)
        _disk_footprint_check(estimated_mb, tmpdir)

        # Run bg and composition HTML concurrently.
        bg_task = asyncio.create_task(_bg())
        comp_html_task = asyncio.create_task(_gen_comp_html())
        comp_html = await comp_html_task
        comp_html_path.write_text(comp_html, encoding="utf-8")
        # Start overlay render as soon as the composition HTML is ready.
        overlay_task = asyncio.create_task(_overlay())
        # Wait for bg.
        bg_path = await bg_task
        bg_path_holder["path"] = bg_path
        # Wait for overlay.
        overlay_path = await overlay_task
        # Final ffmpeg composite.
        if should_cancel and should_cancel():
            raise OverlayRenderError("cancelled before ffmpeg composite", bg_path=bg_path)
        final_path = await _composite(bg_path, overlay_path)
        return final_path
    except OverlayRenderError as exc:
        # If we got a bg path, propagate it.
        if bg_path_holder["path"] is not None and exc.bg_path is None:
            exc.bg_path = bg_path_holder["path"]
        raise
    finally:
        # Clean up the temp composition HTML.
        comp_html_path.unlink(missing_ok=True)
        # Clean up the overlay.mov (the ffmpeg composite produced final.mp4).
        # Leave bg.mp4 — pi_bridge may want to return it.
        (tmpdir / "overlay.mov").unlink(missing_ok=True)
```
